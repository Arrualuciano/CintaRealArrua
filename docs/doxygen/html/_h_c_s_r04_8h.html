Descripción detallada

Driver para sensor ultrasonico HC-SR04.

Implementa la secuencia de medicion mediante una FSM no bloqueante: 

     @code
     HCSR_IDLE
       --> [usuario setea state = HCSR_TRIGGER]
     HCSR_TRIGGER   : emite pulso trigger de 10 us
     HCSR_TRIG_WAIT : espera fin del pulso trigger
     HCSR_ECHO_WH   : espera flanco rising del pin ECHO
     HCSR_ECHO_WL   : mide duracion del pulso ECHO (falling edge)
     HCSR_READY     : distance contiene el resultado en cm
     HCSR_ERROR     : timeout (objeto muy lejos o sin respuesta)
     @endcode

     El driver es independiente del hardware: las funciones de acceso
     al pin y al temporizador se inyectan como punteros a funcion en
     HCSR04_Init().

     Conversion: distancia_cm = tiempo_echo_us / 58

◆ HCSR04_Process()

void HCSR04_Process ( HCSR04_Setting_t * setting)

Ejecuta un paso de la FSM del sensor (no bloqueante).

Debe llamarse frecuentemente desde el main loop. Para iniciar una medicion, setear setting->state = HCSR_TRIGGER antes de llamar a esta funcion. Al terminar, setting->state sera HCSR_READY o HCSR_ERROR.

Parámetros
settingPuntero a la instancia HCSR04_Setting_t.

Ejecuta un paso de la FSM del sensor (no bloqueante).

Debe llamarse frecuentemente desde el super-loop. Para iniciar una nueva medicion, el usuario debe setear:

setting->state = HCSR_TRIGGER;

Al finalizar, el estado sera:

  • HCSR_READY: medicion exitosa, distancia en setting->distance (cm).
  • HCSR_ERROR: timeout, el sensor no respondio.

El usuario debe resetear el estado a HCSR_IDLE despues de leer el resultado para permitir nuevas mediciones.
