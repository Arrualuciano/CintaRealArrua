Descripción detallada

Implementacion del modulo de control de actuadores (FSM).

Ciclo completo de un disparo:

  1. Actuator_Trigger() acepta el disparo --> ACT_WAITING (si delayTicks > 0).
  2. Tras el retardo de espera --> ACT_EXTENDING + activa hardware.
  3. Tras delayTicks ticks en extension --> ACT_RETRACTING + desactiva hardware.
  4. Tras delayTicks ticks en retraccion --> ACT_IDLE.

Cada tick equivale a 2 ms (Timer0 del sistema). El retardo hardcodeado de extension/retraccion es de 75 ticks = 150 ms.

◆ Actuator_Trigger()

uint8_t Actuator_Trigger ( Actuator_t * act)

Solicita el disparo del actuador.

Si el actuador esta en ACT_IDLE, inicia el ciclo de extension. Si esta ocupado en cualquier otra fase, el disparo se ignora.

Parámetros
actPuntero al actuador.
Devuelve
1 si el disparo fue aceptado, 0 si el brazo estaba ocupado.
